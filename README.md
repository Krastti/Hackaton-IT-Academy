# PII Detection Pipeline

Инструмент для автоматического обнаружения персональных данных (ПДн) в файлах датасета. Поддерживает документы, таблицы, изображения и видео. Работает полностью офлайн.

---

## Возможности

- **Широкий охват форматов** — PDF, DOCX, DOC, RTF, HTML, TXT, CSV, XLSX, JSON, Parquet, PNG/JPG/TIFF, MP4 и другие
- **12 категорий ПДн** — паспорт РФ и загранпаспорт, ИНН, СНИЛС, телефон, e-mail, банковская карта, водительское удостоверение, дата рождения, адрес, ФИО, специальные категории (медицина, биометрия, национальность)
- **Валидация данных** — алгоритм Луна для банковских карт, контрольная сумма СНИЛС
- **NER на русском** — извлечение ФИО и топонимов через Natasha
- **OCR** — распознавание текста на изображениях и в видео через EasyOCR (ru + en)
- **Параллельная обработка** — пул воркеров с настраиваемым числом потоков
- **Дедупликация** — по SHA-256 хешу файла и нечёткому сравнению строк
- **Автоматические рекомендации** — уровень защиты (`none` / `low` / `medium` / `high` / `critical`) и действия по нейтрализации рисков
- **Отчёт в CSV** — один файл на запуск с результатами по каждому файлу

## Структура проекта

```
.
├── app.py              # Точка входа, CLI, оркестрация пайплайна
├── Dockerfile          # Docker-образ с предзагруженными ML-моделями
├── requirements.txt    # Python-зависимости
└── src/
    ├── __init__.py
    ├── router.py       # Обход директории, фильтрация, дедупликация файлов
    ├── batcher.py      # Модель данных батча и его жизненный цикл
    ├── extractor.py    # Извлечение текста по формату файла
    ├── scanner.py      # Детекторы ПДн, NER, расчёт уровня защиты
    └── reporter.py     # Запись итогового CSV-отчёта
```

## Быстрый старт

### Через Docker (рекомендуется)

```bash
# Сборка образа (ML-модели EasyOCR загружаются при сборке, ~300 МБ)
docker build -t pii-pipeline .

# Запуск
docker run --rm \
  -v /path/to/dataset:/app/data \
  -v /path/to/reports:/app/reports \
  pii-pipeline --dataset /app/data --output /app/reports --workers 4
```

### Локальный запуск

**Требования:** Python 3.11+, ffmpeg (для видео)

```bash
pip install -r requirements.txt

python app.py --dataset ./data --output ./reports --workers 4
```

## Категории ПДн

| Категория | Описание | Метод |
|---|---|---|
| `passport` | Паспорт РФ (серия + номер), загранпаспорт / MRZ | Regex |
| `inn` | ИНН физического лица (12 цифр) | Regex |
| `snils` | СНИЛС с проверкой контрольной суммы | Regex + валидация |
| `phone` | Номера телефонов в форматах +7 и 8 | Regex |
| `email` | Адреса электронной почты | Regex |
| `bank_card` | Номера банковских карт (алгоритм Луна) | Regex + валидация |
| `driver_license` | Водительское удостоверение РФ | Regex |
| `date_of_birth` | Дата рождения в форматах DD.MM.YYYY и YYYY-MM-DD | Regex |
| `address` | Адреса с ключевыми словами (ул., д., г. и т.д.) | Regex + NER |
| `full_name` | ФИО (2–3 слова) | NER (Natasha) |
| `special_pii` | Медицина, биометрия, религия, национальность | Regex |

## Формат отчёта

Отчёт сохраняется в `<output>/reports_<timestamp>.csv`.

| Поле | Описание |
|---|---|
| `batch_id` | UUID задачи |
| `file_path` | Путь к файлу |
| `file_format` | Расширение файла |
| `categories` | Найденные категории ПДн (через запятую) |
| `total_matches` | Общее число вхождений |
| `protection_level` | Уровень защиты: `none` / `low` / `medium` / `high` / `critical` |
| `recommendations` | Рекомендуемые меры (через `;`) |

### Уровни защиты

| Уровень | УЗ | Число категорий |
|---|---|---|
| `none` | — | 0 |
| `low` | УЗ-4 | 1–2 |
| `medium` | УЗ-3 | 3–4 |
| `high` | УЗ-2 | 5–6 |
| `critical` | УЗ-1 | 7+ |

## Архитектура пайплайна

```
dataset/
   └─ файлы
        │
        ▼
    Router          ← рекурсивный обход, фильтрация по формату,
        │             дедупликация по SHA-256
        ▼
    Batch[]         ← объект состояния для каждого файла
        │
   ┌────┴────┐
   │ThreadPool│      ← параллельная обработка (--workers)
   └────┬────┘
        │
        ▼
   Extractor         ← извлечение текста по формату файла
        │
        ▼
   Scanner           ← детекторы ПДн → уровень защиты → рекомендации
        │
        ▼
   Reporter          ← CSV-отчёт
```
